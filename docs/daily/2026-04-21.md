# 2026-04-21 Daily Change Log

## Amazon Business Integration

- Added Amazon Business Reporting API scaffolding for OAuth start, OAuth callback, and order-line sync Edge Functions.
- Added database tables for Amazon Business connections, OAuth state, and synced order line items.
- Added transaction-level source fields so Amazon Business order lines can appear as reviewable transaction rows without losing the Plaid bank audit trail.
- Added sandbox defaults for Amazon's static Reporting API order-line scenario, including the US marketplace region and sample sandbox order ID.
- Added Amazon Business superseding so Plaid Amazon charges stay as bank audit records but are removed from the normal review queue once Amazon Business line items are available.
- Added Quick Actions for connecting and syncing Amazon Business from the app.
- Added Amazon Business helper tests for endpoint selection, US marketplace region handling, authorization URL construction, and nested Reporting API line-item normalization.
- Added support for constructing the Amazon Business OAuth URL from `AMAZON_BUSINESS_APPLICATION_ID`, avoiding fragile hand-built authorization URLs.
- Added an Amazon Business connection-status Edge Function so the app can show an existing connection and avoid sending an already-authorized account back through Amazon's reauthorization flow.
- Removed direct client-side select access to the Amazon Business connection table so stored refresh tokens stay server-only.
- Clamped Amazon Business sync end dates to the current server time so default same-day syncs do not send a future `orderEndDate`.
- Surfaced Amazon Reporting API status, request ID, and error body details in the sync failure modal.
- Added date-bounded Plaid Amazon superseding so Amazon-looking Plaid rows in the synced Amazon Business date range are hidden from normal review once Amazon line items are imported.
- Added automatic background Amazon Business sync on app load when the connection has not synced recently, while keeping the manual sync button for fallback.
- Added a client-side Amazon/Plaid safety filter so normal review hides Plaid Amazon rows whenever Amazon Business line rows are present, even if server-side cleanup is delayed.
- Made Plaid Amazon server cleanup non-blocking so an update warning cannot fail an otherwise successful Amazon order-line sync.

## Operational Notes

- Amazon Business production order data still depends on completing Amazon's developer profile, identity verification, and Amazon Business Analytics role approval.
- If Amazon requires the EIN/application correction to be handled through a new app client, replace the Supabase Amazon secrets after the corrected client is created.
- Plaid Amazon charges are hidden from normal review after Amazon line items sync for the same date range; the original Plaid rows remain in the database as the bank audit trail.
- Set `AMAZON_BUSINESS_SUPERSEDE_PLAID_MODE=off` only if Amazon Business should stop replacing Plaid Amazon rows.
