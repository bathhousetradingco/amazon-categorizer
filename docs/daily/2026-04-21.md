# 2026-04-21 Daily Change Log

## Amazon Business Integration

- Added Amazon Business Reporting API scaffolding for OAuth start, OAuth callback, and order-line sync Edge Functions.
- Added database tables for Amazon Business connections, OAuth state, and synced order line items.
- Added transaction-level source fields so Amazon Business order lines can appear as reviewable transaction rows without losing the Plaid bank audit trail.
- Added sandbox defaults for Amazon's static Reporting API order-line scenario, including the US marketplace region and sample sandbox order ID.
- Added optional Plaid Amazon superseding behind the `AMAZON_BUSINESS_SUPERSEDE_PLAID=true` Supabase secret. It is intentionally off by default until production Amazon data is verified.
- Added Quick Actions for connecting and syncing Amazon Business from the app.
- Added Amazon Business helper tests for endpoint selection, US marketplace region handling, authorization URL construction, and nested Reporting API line-item normalization.
- Added support for constructing the Amazon Business OAuth URL from `AMAZON_BUSINESS_APPLICATION_ID`, avoiding fragile hand-built authorization URLs.

## Operational Notes

- Amazon Business production order data still depends on completing Amazon's developer profile, identity verification, and Amazon Business Analytics role approval.
- If Amazon requires the EIN/application correction to be handled through a new app client, replace the Supabase Amazon secrets after the corrected client is created.
- Keep Plaid Amazon superseding disabled until a real production sync confirms that Amazon Business line items reconcile to the card charges.
