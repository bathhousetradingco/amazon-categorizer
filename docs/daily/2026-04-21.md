## Edge Functions

- Added `amazon-business-auth-start` to create one-time OAuth state and generate the Amazon authorization URL.

- Added `amazon-business-status` so the frontend can display connection state without exposing refresh tokens.
- Added `amazon-business-sync` to:
  - Refresh Amazon access tokens.

  - Fetch order line items.

  - Create reviewable transaction rows.
  - Hide matching Plaid Amazon rows from normal review.

- Plaid remains the bank transaction source/audit trail.
- Amazon Business is now the source of truth for Amazon purchase detail.
- Plaid Amazon rows are not deleted; they remain in Supabase as original bank evidence.
- Plaid rows that look like Amazon are hidden from normal review once Amazon Business line items exist.
- Added server-side Plaid Amazon cleanup by marking rows with `superseded_at` / `superseded_by_source`.
- Added client-side safety filtering so normal review hides Plaid Amazon rows whenever Amazon Business rows exist, even if server cleanup is delayed.
- Made server-side Plaid cleanup non-blocking so a cleanup warning cannot fail an otherwise successful Amazon line-item sync.
- Future Plaid sync/webhook imports skip Amazon-looking rows once Amazon Business is connected unless `AMAZON_BUSINESS_SUPERSEDE_PLAID_MODE=off` is set.

## Current Expected Behavior

- Opening the app while logged in loads transactions and triggers background Amazon Business sync when stale.
- Amazon Business line items appear as normal reviewable transactions.
- Vague Plaid Amazon bulk charges are hidden from normal review once Amazon line-item rows are available.
- Manual Amazon sync remains available if immediate refresh/debugging is needed.
- If Amazon returns multiple pages, the sync follows `nextPageToken` until all pages are read.
- If the capture audit says `Capture audit passed`, raw Amazon line items, stored line rows, and review rows are aligned for the synced date range.

## Operational Notes

- Keep `AMAZON_BUSINESS_SUPERSEDE_PLAID_MODE` unset for normal operation. Set it to `off` only if Amazon Business should stop replacing Plaid Amazon rows.
- GitHub Actions deploys the static app to GitHub Pages and deploys Supabase Edge Functions on pushes to `main`.
- Database migrations are currently not applied automatically unless `SUPABASE_DB_URL` is added to GitHub repository secrets.
- Original Plaid rows stay in the database for audit/reconciliation even when hidden from normal review.
